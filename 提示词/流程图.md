## 用户管理模块工作流程图 - 绘制规范

### 一、整体要求
- **配色方案**：黑白配色（strokeColor=#000000, fillColor=#FFFFFF）
- **字体规范**：统一 14pt，所有文字粗细一致
- **线条规范**：线宽 1px，所有线条粗细一致
- **布局方式**：垂直纵向布局，从上至下依次排列

### 二、图形组件规范

| 组件类型 | 形状样式 | 尺寸规格 | 使用场景 |
|---------|---------|---------|---------|
| 开始/结束 | 圆角矩形 (rounded=1, arcSize=50) | 宽 100px × 高 50px | 流程起点和终点 |
| 输入/输出 | 平行四边形 (shape=parallelogram) | 宽 150px × 高 50px | 填写信息、信息管理等交互环节 |
| 处理步骤 | 矩形 (whiteSpace=wrap) | 宽 120px × 高 50px | 创建账户、生成令牌、访问资源等操作 |
| 条件判断 | 菱形 (rhombus) | 宽 120px × 高 60px | 数据验证、凭证验证、权限检查 |

### 三、连接线规范

**主流程线（向下）**：
- 样式：edgeStyle=orthogonalEdgeStyle
- 箭头：endArrow=block（平底箭头）
- 圆角：rounded=0（无圆角，直线）
- 连接点：exitX=0.5, exitY=1 → entryX=0.5, entryY=0（底部中心到顶部中心）

**反馈回路（向右折返）**：
- 样式：edgeStyle=orthogonalEdgeStyle, rounded=0
- 箭头：endArrow=block
- 连接点：exitX=1, exitY=0.5 → entryX=1, entryY=0.5（右侧中心到右侧中心）
- 路径：先水平向右 60px，再竖直向上，最后水平向左连接目标
- 竖直段长度：至少保留 45px（约 3 个文字高度），确保文字标签有足够空间

### 四、文字标签规范

**"是"/"否"条件标签**：
- 样式：edgeLabel, html=1, align=center, verticalAlign=middle
- 字体：fontSize=14
- 可连接性：connectable=0
- 位置：relative=1, x=0.5（水平居中）
- 偏移：无偏移（直接在线条上）
- 放置位置：反馈回路竖直段的正中间

### 五、节点间距规范

| 连接类型 | 间距要求 |
|---------|---------|
| 菱形 → 下方矩形 | 40px（保证竖直线段可见） |
| 矩形 → 菱形 | 20px |
| 矩形 → 矩形 | 20px |
| 反馈回路水平段 | 右侧延伸 60px（从菱形右顶点计算） |

### 六、流程结构
 
```
开始 → 填写信息 → 创建账户 → 验证凭证 是→ 生成令牌 → 检查权限 → 访问资源 是→ 信息管理 → 结束
                    ↑         ↓                    ↑         ↓
                    └────否───┘                    └────否───┘
                                                        
```

### 七、关键注意事项

1. 所有 mxCell 的 parent="1"（顶层）
2. 边线必须设置 rounded=0，确保折线无圆角
3. 反馈回路的"是"/"否"标签必须作为 edge 的子元素（parent=边线 ID）
4. 菱形下方竖直线段长度需保证文字标签完全显示且不遮挡线条
5. 所有形状 whiteSpace=wrap 确保文字自动换行
6. 所有形状 html=1 启用 HTML 渲染
7. 如果上一步是交互，则不要菱形判断
8. 条件判断只有是否
9. 是否字样出现在线条出发点
