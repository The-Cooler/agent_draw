# 时序图（四泳道固定）

泳道固定为：**用户**、**前端**、**后端**、**数据库**。每条消息必须落在规定的 **from → to** 上，**不得**把文案画到错误的箭头方向。

## 条数与 `trip`

| `data.trip` | 含义 | `messages` 条数 |
|-------------|------|-----------------|
| `one_way` | 单程（用户→…→用户） | **恰好 7** |
| `double_way` | 双程（再走一轮回到用户） | **恰好 13** |

- 用户描述不足 7 条语义时：由你为 **每一条固定序号** 补写简短文案（仍须满足该条的 from→to），再提交校验。
- 若用户只比单程 **多一点点**（例如勉强凑出 8 条、但未形成完整第二趟）：按 **单程** 处理，**丢弃**第 8 条及以后，只保留 7 条，`trip` 用 `one_way`。
- 若用户明确要 **第二次往返**（例如「再查一次」「二次请求」等完整第二轮）：用 `double_way`，**补全至 13 条**，且第 8–13 条的 from→to **必须**按下表，不可错位。

## 每条消息的固定方向（必须一致）

**单程（序号 1–7）**

| # | from → to |
|---|-----------|
| 1 | 用户 → 前端 |
| 2 | 前端 → 后端 |
| 3 | 后端 → 数据库 |
| 4 | 数据库 → 数据库（内部处理） |
| 5 | 数据库 → 后端 |
| 6 | 后端 → 前端 |
| 7 | 前端 → 用户 |

**双程多出的 8–13**

| # | from → to |
|---|-----------|
| 8 | 用户 → 前端 |
| 9 | 前端 → 后端 |
| 10 | 后端 → 数据库 |
| 11 | 数据库 → 后端 |
| 12 | 后端 → 前端 |
| 13 | 前端 → 用户 |

## 文案规则

- **`text`**：每条 **不超过 8 个字**（按字符数计，与校验一致），非空。
- **`from` / `to`**：字符串必须与上表完全一致（含「数据库」四字）。

## 与 diagram_protocol 的衔接

1. 顶层 JSON：`type` 为 `sequence`，`data` 含 `title`、`trip`、`messages`。
2. `validate_diagram_request` 直至 `valid`。
3. 调用 **`sequence_draw`**，参数：**`project_root`**（与用户消息中的项目根一致）、**`user`**、**`title`**（= `data.title`）、**`trip`**（= `data.trip`）、**`messages`**（= `data.messages`）。不要自拼输出路径。

## `data` 示例（单程）

```json
{
  "title": "登录查询",
  "trip": "one_way",
  "messages": [
    {"from": "用户", "to": "前端", "text": "点击登录"},
    {"from": "前端", "to": "后端", "text": "提交账号"},
    {"from": "后端", "to": "数据库", "text": "查询用户"},
    {"from": "数据库", "to": "数据库", "text": "校验密码"},
    {"from": "数据库", "to": "后端", "text": "返回记录"},
    {"from": "后端", "to": "前端", "text": "下发令牌"},
    {"from": "前端", "to": "用户", "text": "显示首页"}
  ]
}
```

## 模板文件（仓库内）

绘图依赖 Visio 模板（若缺失则工具会报错）：

- `skills/sequence/template/one_way.vsdx` — 消息占位形状文本为 `1. 消息1` … `7. 消息7`
- `skills/sequence/template/double_way.vsdx` — 至 `13. 消息13`

保存结果：`{project_root}/data/{user}/sequence/{title}.vsdx`。
