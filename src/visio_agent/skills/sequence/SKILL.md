# 时序图绘制技能

## ReAct 执行模式（强制）

```
Thought: 理解用户需求，判断流程类型，提取消息
Action: check_message_count - 检查消息条数
Observation: 检查结果
Thought: 如果条数不对，修正消息列表
Action: check_message_length - 检查字数
Observation: 检查结果
Thought: 如果字数超限，修正对应消息
Action: sequence_draw - 绘制时序图
```

**必须按此循环执行，任何检查不通过都要修正后重新检查，直到全部通过才能绘制。**

## 消息映射

### 单程 (one_way) - 7条消息

| 序号 | From -> To |
|-----|------------|
| 1 | 用户 -> 前端 |
| 2 | 前端 -> 后端 |
| 3 | 后端 -> 数据库 |
| 4 | 数据库 -> 数据库（自循环） |
| 5 | 数据库 -> 后端 |
| 6 | 后端 -> 前端 |
| 7 | 前端 -> 用户 |

### 双程 (double_way) - 13条消息

| 序号 | From -> To |
|-----|------------|
| 1 | 用户 -> 前端 |
| 2 | 前端 -> 后端 |
| 3 | 后端 -> 数据库 |
| 4 | 数据库 -> 数据库（自循环） |
| 5 | 数据库 -> 后端 |
| 6 | 后端 -> 前端 |
| 7 | 前端 -> 用户 |
| 8 | 用户 -> 前端 |
| 9 | 前端 -> 后端 |
| 10 | 后端 -> 数据库 |
| 11 | 数据库 -> 数据库（自循环） |
| 12 | 数据库 -> 后端 |
| 13 | 后端 -> 前端 |

## 消息内容要求

### ⚠️ 每条消息不超过8个字

### ⚠️ 根据序号确定消息内容类型

| 序号范围 | 内容类型 |
|---------|---------|
| 1, 2, 6, 7, 8 | 用户/前端/后端之间交互 |
| 3, 5, 9, 12 | 后端/数据库之间 |
| 4, 11 | 数据库自循环（数据校验） |

## 工具调用

### check_message_count
```json
{
  "flow_type": "one_way",
  "messages": ["发送请求", "调用接口", "查询数据", "数据校验", "返回数据", "返回结果", "展示数据"]
}
```

### check_message_length
```json
{
  "messages": ["发送请求", "调用接口", "查询数据", "数据校验", "返回数据", "返回结果", "展示数据"]
}
```

### sequence_draw（两项检查都通过后）
```json
{
  "flow_type": "one_way",
  "messages": ["发送请求", "调用接口", "查询数据", "数据校验", "返回数据", "返回结果", "展示数据"],
  "output_path": "登录时序.vsdx"
}
```

## 错误示例

- 不执行 ReAct 循环 ❌
- 检查不通过仍然绘制 ❌
- 消息内容超过8个字 ❌
- output_path 带斜杠前缀 ❌
